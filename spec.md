# 每日练习验证与恢复趋势 产品设计

**版本**：v0.1
**日期**：2026-07-11

---

## 设计原则

1. 保留暂停实验室三套课程的原始内容、原始顺序与全部必做练习，不用状态评分替代课程。
2. 当天所有必做节点完成后才算完成；一个节点有多个必听音频时，必须逐个听完。
3. 验证层保持轻量：练习前建立基线，节点后记录收获，练习后记录变化与现实行动。
4. 焦虑短期不下降不等于练习无效；趋势同时呈现情绪、身体感受、理解与行动。
5. 恢复记录只保存在本机，并随现有 iCloud 文件备份导出。

## 一、实体与规则体系

### 实体关系

- **课程日**：属于情绪 EBP、CBT 综合或焦虑应对行动计划，包含若干必做节点。
- **节点完成记录**：记录必做节点及其音频是否完成。
- **练习前记录**：每个课程日一条，包含焦虑程度、身体感受、主要触发与今日期待。
- **节点收获**：每个必做节点一条，包含关键收获及其与当下生活的联系。
- **每日总结**：每个课程日一条，包含练习后焦虑、最有帮助节点、新理解与现实行动。
- **恢复趋势**：由已有练习前记录和每日总结自动汇总，不产生重复数据。

### 完成规则

| 规则 | 输入方式 | 示例 |
|------|---------|------|
| 当日完成 | 系统判断 | 全部必做节点完成后标记当日完成 |
| 多音频节点 | 系统判断 | 两段必听音频均达到播放完成阈值后节点才可完成 |
| 节点收获 | 简短文字 | “焦虑是信号，不是命令” |
| 焦虑程度 | 0—10 单选刻度 | 7 分 |
| 身体感受 | 多选 + 自定义 | 心慌、叹气、肩颈紧 |
| 现实行动 | 简短文字 + 状态 | “今晚不查看项目群”／已做到 |

## 二、用户故事

### 场景一：开始当天练习

**US-1.1 建立练习前基线**

作为正在恢复中的练习者，当我开始当天课程时，我希望快速记录此刻状态，以便之后判断练习带来的即时变化。

- 可选择 0—10 的焦虑程度。
- 可多选常见身体感受，并补充其他感受。
- 可填写主要触发和今日期待。
- 已保存内容再次进入时能够继续编辑。

### 场景二：完成课程节点

**US-2.1 完整完成必听内容**

作为希望认真完成课程的练习者，当节点包含一个或多个必听音频时，我希望系统准确判断每段音频是否听完，以免遗漏内容。

- 每段必听音频独立记录播放完成状态。
- 重复播放同一音频不会误判其他音频已经完成。
- 未听完全部必听音频时，不能提前完成该音频节点。
- 纯阅读节点仍由练习者主动确认完成。

**US-2.2 验证节点收获**

作为练习者，当我完成一个节点后，我希望用很短的记录提炼收获，以便把课程知识与今天的处境连接起来。

- 每个节点可填写“我理解到什么”。
- 每个节点可填写“它与我今天有什么关系”。
- 内容自动保存到当前课程日并可修改。
- 收获记录不改变暂停实验室原始课程内容。

### 场景三：结束当天练习

**US-3.1 完成每日总结**

作为练习者，当当天全部课程节点完成后，我希望记录练习后的状态与一个现实行动，以便验证学习是否进入生活。

- 可记录练习后焦虑程度。
- 可选择当天最有帮助的节点。
- 可填写新的理解和一个现实行动。
- 后续可将现实行动标记为已做到或未做到。

### 场景四：回顾恢复变化

**US-4.1 查看近期恢复趋势**

作为练习者，当我想了解课程是否对自己有帮助时，我希望看到近期记录的整体变化，以便基于事实复盘而不是只凭当天感觉判断。

- 汇总最近七个有状态记录的课程日。
- 同时显示练习前后焦虑、身体感受和现实行动完成情况。
- 明确提示一次练习后焦虑未下降不代表练习无效。
- 没有足够记录时引导从下一次练习前开始记录。

### 场景五：备份与恢复

**US-5.1 保护恢复记录**

作为练习者，当我使用现有 iCloud 文件备份时，我希望新的状态和收获记录一同备份，以免换设备或清缓存后丢失。

- 新记录保存在现有课程日数据中。
- 创建备份后包含练习前、节点收获和每日总结。
- 恢复旧备份时不要求存在新字段。
- 新记录发生变化后显示“有新内容待备份”。
