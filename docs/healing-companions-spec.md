# 练习精灵图鉴产品设计

**版本**：v1.5
**日期**：2026-07-17

## 设计原则

1. 每天完成练习后发现一只精灵，让坚持被具体看见。
2. 每只精灵的技能严格对应当天课程能力，不承诺治疗或消除症状。
3. 文案温和、具体、可理解，不制造必须马上好起来的压力。
4. 精灵是课程知识的记忆线索，不能替代重点知识与个人收获。
5. 角色拥有昵称、生活地点与朋友关系，让收集具有连续故事感；关系只提供温暖陪伴，不制造社交压力。
6. 田田是练习者和图鉴主人；缓缓是生活伴侣与陪伴角色，不被描述为焦虑练习或疗愈对象。
7. 图鉴详情以视觉美感与阅读舒适为最高优先级，不为强行塞进一屏而压缩内容；卡片内不设置折叠交互，所有故事自然向下展示，保存图片时始终输出完整内容。

## 一、实体与规则体系

- **每日课程**：提供主题与重点知识。
- **疗愈精灵**：与课程天数一一对应，包含昵称、称号、技能、性格、喜好、居住地和原创插画。
- **精灵关系**：每只精灵连接同课程的前后伙伴，以及另一课程的一位笔友，形成完整关系网。
- **成果卡**：记录用户当天的个人收获和心态，并解锁对应精灵。
- **图鉴**：聚合 67 只精灵与三个课程的收集进度。
- **主人公**：田田完成练习并收藏精灵；缓缓作为妻子出现在生活故事中，为田田提供自然陪伴。

| 规则 | 输入方式 | 示例 |
|---|---|---|
| 精灵名称 | 固定配置 | 微光芽 |
| 专属技能 | 固定配置 | 小步生长 |
| 技能帮助 | 固定配置 | 看见已经做到的部分，减轻完美压力。 |
| 解锁条件 | 系统判断 | 完成当天全部任务并保存成果卡 |
| 历史兼容 | 自动匹配 | 已解锁成果卡按课程和天数自动获得精灵 |
| 原创插画 | 课程 + 天数唯一映射 | EBP-01 与 CBT-01 使用不同角色和图片 |
| 生活资料 | 固定配置 | 住在微光花园、喜欢清晨的露珠 |
| 朋友关系 | 系统按顺序生成 | 两位日常搭档 + 一位跨课程笔友 |
| 主人公故事 | 课程 + 天数动态生成 | 田田的练习、两人日常、精灵寄语 |
| 卡片图片 | 用户主动保存 | 生成包含插画、资料、完整故事与课程重点的 PNG |

## 二、用户故事

### 场景一：完成当天练习

**US-1.1 发现今日精灵**

作为正在恢复的练习者，当我完成今天的课程并记录收获时，我希望发现一只与今天内容对应的精灵，以便感受到完成和积累。

- 一天只对应一只固定精灵。
- 精灵技能与当天重点知识一致。
- 不要求额外填写内容。

### 场景二：查看图鉴

**US-2.1 回顾已获得的能力**

作为练习者，当我打开图鉴时，我希望先看到精灵名字与技能，以便迅速回忆自己已经练过哪些能力。

- 缩略卡展示名字、课程主题和技能名。
- 未解锁卡不透露名字和技能。
- 三个课程可分别筛选并查看进度。

### 场景三：打开成果卡

**US-3.1 理解技能怎样帮助自己**

作为焦虑状态中的练习者，当我点开一只已获得的精灵时，我希望看到简短具体的技能帮助，以便把课程知识连接到现实生活。

- 详情依次展示精灵、技能帮助、课程重点、个人收获和心态。
- 技能帮助不使用医疗承诺。
- 暗色模式下保持清晰可读。

### 场景四：查看历史成果

**US-4.1 自动补全历史精灵**

作为已经完成过课程的练习者，当功能更新后，我希望过去的成果卡自动出现对应精灵，以便不需要重新练习或重新填写。

- 使用课程编号和天数匹配。
- 不改变原有个人记录。
- 不改变原解锁日期。

### 场景五：认识精灵小世界

**US-5.1 在成果卡中认识角色**

作为正在坚持练习的人，当我打开已解锁的成果卡时，我希望看到精灵的昵称、生活资料和朋友关系，以便让完成练习的纪念更像一次有故事的角色相遇。

- 昵称亲切、原创、易记，不使用已有商业角色名称。
- 卡片展示称号、居住地、性格、喜欢的事和朋友关系。
- 关系文字连接同课程伙伴与跨课程笔友，所有 67 只精灵都处于关系网中。
- 生活设定不遮挡课程技能、今日重点和个人收获。
- 暗色模式下资料卡与关系文字保持清晰可读。

**US-5.2 解锁不同的角色**

作为练习者，当我完成新的课程日时，我希望遇到全新的昵称和插画，而不是旧角色换色或重复出现，以便每次解锁都有独立的收集价值。

- 三个课程共 67 天，必须对应 67 个不同昵称与 67 张不同插画。
- 图片保持统一的温柔 3D 毛绒风格，但角色轮廓、物种或道具有明显区别。
- 图片不含文字、品牌、水印或医疗承诺。
- 历史成果按课程和天数自动映射到新版角色。

### 场景六：田田练习，缓缓陪伴

**US-6.1 在获得页读到当天的相遇故事**

作为练习者田田，当我完成当天任务并获得精灵时，希望看到精灵为何到来，以及缓缓如何自然地出现在生活故事中，以便图鉴既记录我的练习，也保留两个人的温暖日常。

- 获得页展示田田的相遇故事、缓缓的生活陪伴和精灵寄语。
- 每天的描述随课程、天数、精灵技能、住处和爱好变化。
- 图鉴详情页保留同样的完整故事，方便日后回看。
- 图鉴总览明确显示为“田田的练习图鉴”。
- 缓缓不被描述为完成焦虑练习、接受疗愈或拥有焦虑症。
- 主人公名称不与精灵昵称重复，避免角色身份混淆。

### 场景七：舒适浏览与保存卡片

**US-7.1 在手机上舒适浏览完整张卡**

作为图鉴主人田田，当打开一张已解锁卡片时，希望首先看到比例舒展、排版稳定的角色卡，并通过自然滚动继续浏览资料，以便收藏过程更有仪式感。

- 插画、留白、字号与资料块保持舒展协调，不为一屏展示牺牲卡片美感。
- 完整相遇故事、日常、朋友关系与寄语直接展示在卡片内，不折叠、不隐藏。
- 小高度屏幕只适度调整插画，不隐藏已经收藏的信息。
- 整张卡片允许页面自然滚动，不裁切任何内容。

**US-7.2 保存完整卡片图片**

作为图鉴主人田田，当喜欢一张精灵卡时，希望保存一张完整图片，以便放进相册或与缓缓分享练习成果。

- 卡片详情下方提供明确的“保存完整卡片图片”按钮。
- 输出 PNG 包含精灵插画、基础资料、完整故事、朋友关系、寄语和课程重点。
- 手机优先打开系统分享面板，其他环境直接下载图片。
- 图片绘制不依赖外部服务，离线状态仍可使用。
- 生成失败时保留当前页面并给出温和提示。
